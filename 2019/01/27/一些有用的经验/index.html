一些有用的经验

PAT

  • 即使是C++,输入输出选择printf和scanf,效率更高; 除非对于字符串string容器,才可能使用;

  • 无论是C++还是C语言,选择以.cpp文件进行保存,以C++标准进行编译.(防止一些混用)

  • 数据类型大致范围:

    • int:-2*10^9~2*10^9
    • long long int:-9*10^18~9*10^18,对于long long型变量,赋值常量时,需要加后缀LL; long long a=12345688LL;

  • C语言中无bool类型,C99之后有<stdbool.h>中定义bool类型; C++中有bool类型; C中可以通过宏定义进行自行定义;

  • ASCII码中,小写字母ASCII码值比大写字母ASCII码大32; \0:空字符,ASCII码值为0;

  • scanf对于数据的读入结束以空格或者换行为标志;(字符数组也是一样,遇到空格/回车则截止,不将空格/回车作为字符串内容,然后添加’\0’在末尾)

  • gets读取字符串只以换行作为结束标志,遇到空格继续读入,不将回车作为字符串内容,然后添加’\0’在末尾;
  • cin读取字符串以空格或者换行作为结束标志;
    • scanf没有读取回车和空格,因此读取字符串之后,回车依然存在;因此在scanf后经常加getchar来吞掉多余的回车/空格;
    • gets读取了回车,只不过不将其作为字符串的内容,不需要加getchar;
    • cin读取了回车,不会将其作为字符串内容;

  • printf和puts在输出字符串只会在遇到’\0’时停止输出;(puts会将’\0’转换为’\n’然后进行输出,比pritnf多一个换行)

  • <math.h>的函数:pow(double r,double p),sqrt(double x),log(double x),round(double x);

  • 如果数组较大(10^ 6级别),需要将其定义在主函数之外,否则会出现异常.因为局部变量的空间来自于栈,允许申请的空间不大;全局变量来自于静态存储区,能够分配的空间较大.

  • 数组初始化:memset(数组名,值,sizeof(数组名)); 为每一个数组元素赋一个相同的值. 需要<string.h>. sizeof(数组名):返回的是数组的大小. memset的机制是按字节赋值,因此最好是赋值0/-1. 中有fill函数进行初始化(memset速度快)

  • 字符串函数:需要<string.h>

    • strlen(字符数组):返回第一个’\0’前的字符个数
    • strcmp(字符数组1,字符数组2):比较两个字符数组(比较规则:从对应的第一个元素开始,用ASCII码进行比较,如果字符数组1的ASCII码小于字符数组2的ASCII码,那么返回负整数;如果大于,那么返回正整数;否则比较第二个元素,如果全部相同,那么返回0)
    • strcpy(字符数组1,字符数组2):将字符数组2赋值给字符数组1,复制包括’\0’;
    • strcat(字符数组1,字符数组2):将字符数组2接到字符数组1(的’\0’)之后;

  • sscanf和sprintf:字符串处理

    • sscanf(str,”%d”,&n); :将str中的数以%d的形式存储到n中(前提是str的数据符合规范);
    • sprintf(str,”%d”,n); :将n以%d的形式存储到字符数组中;

  • 对于int 指针(以p为变量),p+1指向的是下一个int型(byte+4);对于double 指针,p+1指向的是下一个double(byte+8),因此指针的运算一般是放在数组中(能够保证数据类型的一致性); (指针跨越的是数据个数,而不是字节个数); 指针类型是一个unsigned整型;

  • 对于结构体的定义,无法在其内部定义自己,但是可定义自身的指针:

  • 结构体也能够使用构造函数,跟类一样;
    • 具有默认构造函数: node(){}; 默认值赋值
    • 能够使用成员初始化列表(member initialization list)
    • 只要自己构造过构造函数,那么必须重新声明默认构造函数,否则,默认构造函数不起作用.

  • cin将一行内容读入char中: char str[100]; cin.getline(str,100);

    • cin读入字符串以空格作为结束,如果要读入带空格的字符串,可以使用getline(istream&,string); getline:将流的内容读入字符串中
      • istream& getline (istream& is, string& str);

  • cin将一行数据读入string容器内: string str; getline(cin,str);

  • 浮点数的比较:因为浮点数的存储不精确,然而一些比较操作非常精确,因此需要使用一些特殊的手段:引入一个极小数eps来对误差修正,通常取eps=1e-8

    • 对于==,如果b-eps<=a<=b+eps,那么==, const double eps=1e-8; #define Equ(a,b) ((fabs((a)-(b)))<(eps))
    • 对于>,a>b+eps;(b-eps~b+eps是b的不精确范围,因此,如果大于b,那么就需要大于最大)(一定要使每一个元素都符合)
    • 对于<,a< b-eps
    • 对于>=,如果a>=b-eps(因为包含等号,所以要包含b的范围)
    • 对于<=,如果a<=b+eps

  • π：通过acos(-1.0)得到

  • 一般OJ的运算速度为10^7~10^8次,需要预估时间复杂度; (一般来说,空间复杂度够用;时间复杂度可能不够,常常采取空间换时间)

  • 黑盒测试:系统准备数据,提交程序运行,将得到的结果与标准答案进行字符串的比较;

    • 单点测试:系统根据每组数据的输出结果给出分值(即使不是全对,也能有部分的分值) (PAT) (程序从开始到结束只需要运行一组数据,一组数据对应一次程序的运行,这也是能够部分得分的原因,所以可以不需要加while(scanf))
    • 多点测试:要求运行所有数据,输出所有结果必须全部正确,才算通过.(要求一个程序一次必须运行所有的数据)
      • 输入要求: 输入数据是一个文件,通常需要反复读取,while(scanf!=EOF)/while(gets()!=NULL)/while(cin>>s)

  • C++允许这样定义数组:

  • printf输出格式:

    • u:以无符号十进制进行输出; %u,%lu,%llu;
    • .nlf:表示小数点后保留n位;

  • 注意特殊值的处理;

  • 字符数组的比较: char a[5]="Hell"; char b[5]="Hell";

    • 如果使用a==b,比较的不是内容,而是地址,因此a==b得到的结果是false(不相等);
    • 如果使用strcmp(a,b),得到的结果是0,表示的是相等;

  • 慎用一些工具,容易超时:

    • cin,cout:容易超时;
    • string比char[]容易超时,但是方便,string只能搭配cout和cin;

  • (signed) int的数据范围:大于10^9小于10^10;
