类数据成员不能指定为自身类型，但是可以指定为引用或者指针类型

C++

原因:C++要求，当实例化一个类的对象时，编译器会根据类的定义来分配相应的存储空间。
也就是说，在创建对象前，一定要有完整的类定义，这样编译器才能正确的计算所需空间。如果没有完整的定义，相当于一种无限初始化（递归）。

对于指针和引用，他们的存储空间大小与指向的类型无关，因此可以正确地分配内存空间。

实例化类的对象的步骤：

  1. 先构造成员变量。
  2. 然后调用自身的构造函数。

面向对象的分析

软件工程

前面讲到，面向对象包括：OOA，OOD，OOP，现在来考虑OOA

OOA的目的：分析问题，寻找到对象，分析对象的关系，忽略与实现有关的问题，建立系统的对象模型。
OOA的工具：用例图，类图，时序图，协作图，状态图。
OOA的过程：先通过需求分析建立需求模型（功能模型，从用户的角度获取功能需求），然后通过分析问题，发现对象和类，得到分析模型（静态结构模型：类图，动态结构模型：状态图/顺序图）

建立静态模型

过程：根据需求模型，识别分析类；确定类的属性，方法，关系；绘制类图

分析类：是一种在概念上较为抽象的类，只有应用逻辑有关（应该会派生出很多子类）。
分析类的分类：实体类（存储信息），边界类（参与者和系统的交互），控制类（运行的控制逻辑，负责协调实体类和边界类）。
分析类之间的关系属于关联关系。

面向对象基本概念

软件工程

面向对象的历史

在面向对象之前，一直使用结构化分析方法（面向过程）。

结构化设计方法的特点：

  • 以功能为核心进行设计，把软件看成处理数据的流。（功能和数据流）
  • 自顶向下，将问题逐步分解，从而实现系统功能。
  • 数据结构由算法决定。
  • 结构化软件=数据结构+算法。
  • 通过全局变量保存系统的状态。

C语言开发的软件一般属于结构化软件。

结构化分析的缺点：难以应付变化。

面向对象的特点（与面向过程相比）

系统看作对象的集合，每个对象包含一组表示自身特性的数据和数据上的操作集合。考虑问题的核心从过程转移到实体对象。

系统的状态，算法，数据分布在各个用户上。

对象之间的通信通过消息进行。

面向对象软件=对象+消息。

面向对象的优点：

  • 更加适合于解决实际问题
  • 容易进行变更

面向对象包括：OOA（面向对象分析），OOD（面向对象设计），OOP（面向对象编程）

面向对象基本概念

对象（object）：描述客观事物的实体，拥有属性（attribute）和操作（operation）。 例如：一辆车，属性：价格，重量等； 方法：启动，停止，加油等。

类（class）：具有相同属性和操作的对象的集合，是一种抽象的模型。 比如，所有的汽车可以构成一个汽车类。 想要完整的描述类，可以用类图（class diagram）进行静态化描述。

对象是动态的，类是静态的。

类的属性（attribute）：有三种类型，public，private，protected。

消息（message）：一个对象向其他对象发出的请求。 对象通过消息连接在一起，构成了完整的系统。

封装（encapsulation）：隐藏对象的属性，仅对外提供接口来进行属性的访问和修改。（不让外界直接调用对象的属性，但是提供API接口让外界调用）封装的目的在于增强安全性。

接口（interface）：接口定义的是一组属性和操作的描述，而不是操作的实现，需要用其他类来实现它定义的操作。接口的意义体现在设计当中，Java语言用的很多。
另外还有一个抽象类，抽象类允许在普通的类中添加抽象方法。抽象方法就是只有定义，没有实现的方法。抽象类自己不能定义实例，必须依靠子类实现进行定义实例。抽象类与接口类似，只不过接口是所有的方法都为抽象方法，抽象类是一部分方法是抽象方法。

继承（inheritance）：子类可自动拥有父类的所有属性和操作。继承分为单一继承和多重继承。

多态性（polymorphism）：同一个操作作用于不同对象上有不同的行为，产生不同的结果。
静态多态性：一个类或一个函数中的同名函数，它们根据参数表（类型以及个数）区别语义。
动态多态性：在一个类层次的不同类中的重载函数，它们一般具有相同的函数，因此要根据指针指向的对象所在类来区别语义。

对象之间的五类关系

继承泛化关系：表示了对象的一般/特殊关系，一般通过继承实现。 比如：动物与猫。 （is a kind of）

聚合（aggression）/组合（composition）关系：表示了对象的组成关系。
聚合和组合的区别：聚合要求整体与部分的生命周期独立，例如公司和职员，课程和学生。组合要求整体与部分的生命周期统一，例如人和心脏。 (组合：is part of，聚合：own a)

关联关系：表示对象之间的静态联系，通过对象的属性的联系显现。对象实例的连接称为链接（link），存在实例连接的对象类之间的联系称为关联（association）。 关联具有方向性：单向关联，只有一方知道关联关系；双向关联，双方都知道关联关系。 关联数目比有1：1，1：n，n：1，m：n，通常m：n的关系需要添加一个关联类来存储关系信息。 (has a)

聚合关系和关联关系：聚合关系可以看成是一种特殊的关联关系，只不过聚合关系的关联程度大（比较抽象，没太大影响），聚合关系可写成关联关系（这种关系主要是在类图中体现，软件工程比较严格，需要先描述，再进行编程。）聚合关系强调地位不对等（买家和所买的物品），而关联关系强调双方地位比较接近（卖家，卖家）

依赖关系（dependency）：表示对象的消息联系，是一种动态联系。一个对象需要另一个对象的服务（属性/方法），做出请求，并且等待对方的响应信息。它是一种很弱的联系关系。 （use a）

区分依赖关系和关联关系：
依赖关系主要是通过函数进行联系，在函数中引用了其他对象，关系的范围是局部性的。
关联关系是通过属性进行联系，作用范围是全局性的（整个对象）。

实现关系（realization）：用于描述接口和其实现类的关系。

对象联系的强度： 继承 >>> 组合 >> 聚合 >> 关联 >> 依赖

DFS

Practice

DFS:深度优先搜索.

在我看来,DFS的特点如下:

  1. 讲究层次性,在写DFS的时候,一定要弄清楚什么东西代表DFS的层次; 除此之外,弄清楚DFS的结束规则;(递归式+结束规则)
  2. DFS是思想,栈是工具(记录某些信息),递归是手段;
  3. 在写DFS的时候,牢记:在本层需要模拟所有情况(多个递归函数,参数不同),在递归返回的时候,需要回到初始状态,以便模拟所有情况;这也体现在其书写特点;
  4. DFS需要考虑所有情况,虽然可以通过剪枝来提高实际运行效率,但是还是尽量少使用;
  5. DFS处理的问题常与序列顺序,存在性问题(存在,找出最优解/不存在)有关;
  6. DFS就是树的先序遍历过程;(等价)

这是codeup_contest10000608组合数的题目,最明显的特点就是在DFS内进行下一次递归的时候,前后进行相反的操作,目的是为了恢复,方便下一次DFS;

这是PATA1051,有一例超时了,因为我的方法是通过DFS记录所有可能结果,并遍历;因此超时也是很正常的; 不过DFS书写的方法还是不变;

完全背包问题

algorithm

问题描述:有n件物品,每件物品重量w[i],价值c[i],选取若干件物品放入容量为N的背包中,每件物品可选择,求使得价值最多的方案?

方案1:使用DFS,与01背包的区别:

  • 在01背包问题中,每一件物品考虑之后就考虑下一件物品,以物品序号作为DFS的层次;(重量也是一个限制条件)
  • 在完全背包中,需要继续考虑该物品; 因此,DFS的层次划分并不是完全根据物品序号(也受最大物品序号的限制),只要重量不超过额定重量(重量超过额定重量,这就是该DFS的死胡同),DFS就可以继续.

01背包问题

algorithm

问题描述:有n件物品,每件物品重量w[i],价值c[i],选取若干件物品放入容量为N的背包中,每件物品仅选一次,求使得价值最多的方案?

方案1:对于小规模的问题,可以采用DFS;对于每一个物品都有两种选择,然后DFS的最大层数为n;从而形成DFS. (O(2^n),例举所有可能性)

对于DFS这种寻求所有可能性的算法,通常可以搭配剪枝(去除一些不合理的情况),来减少算法的时间. (剪枝在理论上/数学上不会减少时间复杂度,但是在实际运行上,效果显著)

codeup:任务调度

该题的输入有个隐藏的选项,已经出现过的前序不可以出现在之后的后序中; 因为有这个,题目感觉还是容易了很多;否则,我也不知道该怎么做;

  • 这个题关键在于排序(拓扑排序),排序的方式有两种,优先的是拓扑关系,其次是字典序;构建一个结构体,包含了两个排序的元素以及排序的方法;
  • 通过unordered_map来实现taskname和level的对应关系;
  • 通过priority_queue和重载的<来实现排序;
  • task的level构造原则:
    • 对于第一次出现的前序,那么设置level=0 (最高优先级)
    • 对于第一次出现的从序,那么设置level为其前序的level+1 (紧随其后)
    • 对于第二次及以上出现的从序,那么设置需要比较已经记录的level和该次前序level+1的值,取其中最大的值作为该task的level
    • 如果两个task的level相同,则它们的排序通过字典序排序;(在拓扑关系上同级)
    • level小的task先进行输出
