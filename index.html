DFS

Practice

DFS:深度优先搜索.

在我看来,DFS的特点如下:

  1. 讲究层次性,在写DFS的时候,一定要弄清楚什么东西代表DFS的层次; 除此之外,弄清楚DFS的结束规则;(递归式+结束规则)
  2. DFS是思想,栈是工具(记录某些信息),递归是手段;
  3. 在写DFS的时候,牢记:在本层需要模拟所有情况(多个递归函数,参数不同),在递归返回的时候,需要回到初始状态,以便模拟所有情况;这也体现在其书写特点;
  4. DFS需要考虑所有情况,虽然可以通过剪枝来提高实际运行效率,但是还是尽量少使用;
  5. DFS处理的问题常与序列顺序,存在性问题(存在,找出最优解/不存在)有关;
  6. DFS就是树的先序遍历过程;(等价)

这是codeup_contest10000608组合数的题目,最明显的特点就是在DFS内进行下一次递归的时候,前后进行相反的操作,目的是为了恢复,方便下一次DFS;

这是PATA1051,有一例超时了,因为我的方法是通过DFS记录所有可能结果,并遍历;因此超时也是很正常的; 不过DFS书写的方法还是不变;

完全背包问题

algorithm

问题描述:有n件物品,每件物品重量w[i],价值c[i],选取若干件物品放入容量为N的背包中,每件物品可选择,求使得价值最多的方案?

方案1:使用DFS,与01背包的区别:

  • 在01背包问题中,每一件物品考虑之后就考虑下一件物品,以物品序号作为DFS的层次;(重量也是一个限制条件)
  • 在完全背包中,需要继续考虑该物品; 因此,DFS的层次划分并不是完全根据物品序号(也受最大物品序号的限制),只要重量不超过额定重量(重量超过额定重量,这就是该DFS的死胡同),DFS就可以继续.

01背包问题

algorithm

问题描述:有n件物品,每件物品重量w[i],价值c[i],选取若干件物品放入容量为N的背包中,每件物品仅选一次,求使得价值最多的方案?

方案1:对于小规模的问题,可以采用DFS;对于每一个物品都有两种选择,然后DFS的最大层数为n;从而形成DFS. (O(2^n),例举所有可能性)

对于DFS这种寻求所有可能性的算法,通常可以搭配剪枝(去除一些不合理的情况),来减少算法的时间. (剪枝在理论上/数学上不会减少时间复杂度,但是在实际运行上,效果显著)

codeup:任务调度

该题的输入有个隐藏的选项,已经出现过的前序不可以出现在之后的后序中; 因为有这个,题目感觉还是容易了很多;否则,我也不知道该怎么做;

  • 这个题关键在于排序(拓扑排序),排序的方式有两种,优先的是拓扑关系,其次是字典序;构建一个结构体,包含了两个排序的元素以及排序的方法;
  • 通过unordered_map来实现taskname和level的对应关系;
  • 通过priority_queue和重载的<来实现排序;
  • task的level构造原则:
    • 对于第一次出现的前序,那么设置level=0 (最高优先级)
    • 对于第一次出现的从序,那么设置level为其前序的level+1 (紧随其后)
    • 对于第二次及以上出现的从序,那么设置需要比较已经记录的level和该次前序level+1的值,取其中最大的值作为该task的level
    • 如果两个task的level相同,则它们的排序通过字典序排序;(在拓扑关系上同级)
    • level小的task先进行输出

STL-priority_queue

Practice

基础定义

  • priority_queue:优先队列,底层通过堆(完全二叉树,大根堆/小根堆)实现.(本质是堆)
  • 特点:队首元素优先最高,然后依次降低.(我认为是对堆的一种抽象表示)
  • 头文件:
  • 定义:priority_queue name;

priority中关于优先级的设置(在定义中实现)

  • 基本数据类型的优先级的设置:(int,double等)

    • 一般写法:priority_queue q; 数字大的优先级越高
    • 完整写法:priority_queue<int,vector,less > q;
      • less后需要加空格,再加>;(预防编译错误)
      • 第一个int表示优先队列的元素
      • 第二个vector表示承载堆的容器,vector的数据类型跟随第一个参数的类型
      • 第三个less表示比较类,less表示数字越大优先级越大 ,greater表示数字越小优先级越大

  • 结构体的优先级设置:

    • 结构体需要搭配重载函数,只能重载小于号,不能重载大于号,会产生编译错误;(不过能通过重载小于号能实现和重载大于号一致的功能)
    • 优先队列以结构体作为元素,在建堆的比较中,和基本类型一样需要对元素作为比较,不过结构体不能直接比较,需要进行重载来实现对结构体的比较;
    • 重载<的规则:如果返回值是true,则第二个数优先级高;返回值是false,则第一个数优先级高.
      • 如果重载函数内部实现的是<的功能,如果f1<f2,返回true,f2的优先级高,值大的优先级高;
      • 如果重载函数内部实现的是>的功能,如果f1>f2,返回true,f2的优先级高,值小的优先级高;
    • 如果结构体比较大,重载函数的参数使用引用提高效率;(搭配const) (const fruit &f1,const fruit &f2);
    • 结构体也能够采用完整定义, priority_queue<fruit,vector<fruit>,cmp> q; (cmp的定义见代码中)

函数

  • top():返回堆顶元素. (没有front()和back())
  • push(x):令x入队. O(logN)
  • void pop():删除堆顶元素. O(logN)
  • empty():检测队列是否为空,true则空,false则非空. O(1)
  • size():返回队列元素个数. O(1)

STL-queue

C++

基础定义

  • header:
  • 定义: queue name;

函数

  • front()访问队首元素. O(1)
  • back()返回队尾元素. O(1)
  • push(x):将x入队. O(1)
    • 标准形式:void push (const value_type& val); 参数是一个reference to const(常量引用),因此不能改变queue中元素的值; (如果想要改变队列中的元素,那么队列中的元素可以存放数组的下标,通过对数组元素的值改变,达到目的)
  • pop():令队首元素出队并作为返回值. O(1)
  • empty():检测队列是否为空,true则空,false则非空. O(1)
    • empty经常搭配pop()和front()等函数,预防空队列取元素;
  • size():返回队列元素个数. O(1)

常见数据类型的转换

Practice

int转换string

字符串流操作

  • 通过字符串流进行操作:istringstream,ostringstream,stringstream ; 需要头文件
  • 字符串流在进行多次转换时,每次读入数据不会将上次缓冲区的内容覆盖掉,而是增添到尾部;如果需要清除缓冲区,需要使用clear();
  • 在C++中,一般采用库进行数据转换,更加安全,可通过重载来支持各种类型的转换.

string转换char[]

通过str.copy()

size_t string::copy(char* s,size_t len,size_t pos=0)
pos:字符串的起始位置,默认为0;
len:选取字符串的长度;
返回值:实际选取字符串长度,一般和参数len相等,除非pos+len超过字符串的长度;
特点:不会为数组添加’\0’,需要手动添加’\0’;

通过strcpy()

const char* c_str() const noexcept;
将字符串转换成为字符数组,在数组后面添加’\0’;

char strcpy(char destination,const char* source)
strcpy是C Library的函数,操作数为两个char[];
特点:source的所有内容(从开始到终止符’\0’,包括终止符)传给destination;

char[]转换为string

C++提供低级向高级转换的服务;
